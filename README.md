# LLM Sentinel — 上下文安全哨兵

在 LLM 请求前检测用户输入中的不当内容，向主对话模型注入安全警示。

## 功能

| 模块 | 说明 |
|------|------|
| **性暗示检测** | 检测性暗示、色情引导、非安全内容（含错字/谐音/拆字等绕过手段） |
| **角色状态覆写检测** | 检测用户替 AI 角色决定动作、表情、心理、生理状态的行为 |
| **时间线覆写检测** | 检测用户强制操控对话时间线（跳过时间跨度、改写场景时间等） |
| **对话历史参考** | 可选参考近 N 轮对话，识别渐进式越界与连续覆写模式 |

检测到问题后，向主对话模型的系统提示词中注入对应的警示规则，引导模型拒绝不当请求。

## 工作流程

```
用户输入 → 拉取对话历史（可选） → 哨兵模型分析 → 解析结果 → 注入警示 → 主模型生成回复
```

多条规则使用 `asyncio.gather` 并行检测，共享一次延迟。

## 对话历史

哨兵模型可参考近期对话历史，用于识别单轮检测难以覆盖的连续模式：

- **渐进式越界**（grooming）：每条单独看都"无害"，但连续多轮逐步升级
- **状态覆写模式**：覆写型用户通常每轮都写括号动作，连续看比孤立一句更可靠

### 数据来源

| 来源 | 隔离方式 | 说明 |
|------|----------|------|
| **AstrBot 自带上下文**（默认） | 群聊共享 | 从 `conv.history` 读取 |
| **chat_memory 插件**（可选） | 按用户 ID 隔离 | 群内每个用户独立历史，需先安装该插件 |

> 启用 chat_memory 前需先安装该插件，否则自动回退。

## 配置

### 基本设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 哨兵模型 | 空（用当前对话模型） | 参考历史时建议较大模型，仅当前输入可用轻量模型 |
| 使用 chat_memory | 关闭 | 启用后按用户 ID 隔离读取历史 |

### 日志设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 日志附加机器人ID | 关闭 | 多 Bot 环境区分 |
| 调试日志提级 | 关闭 | debug 日志以 info 级别输出 |

### 检测规则

每条规则独立配置，内置三类模板（性暗示 / 状态覆写 / 时间线覆写），也支持自定义。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 最小检测长度 | 0 | 本规则的最小检测长度（0=不启用，全部检测） |
| 参考对话轮数 | 0 | 本规则可参考的近期对话轮数（0-10），推荐 3-5 轮识别连续模式 |
| 检测提示词 | — | `{text}` 替换为用户当前输入；`{history}` 替换为近期对话历史块（未配置轮数时为空） |
| 警示模板 | — | `{detail}` 替换为哨兵检测到的具体涉嫌疑句 |

## 适用场景

- 防止「喂菌子」（逐步污染 LLM 上下文）
- 拦截角色状态覆写（如 `你脸红了`、`她靠近了你` — 用户替 AI 角色写动作/心理，不限符号格式）
- 阻止时间线操控（如 `一个月后我们已经是恋人了`）
- 阻止性暗示/色情引导（含错字/谐音/拆字等绕过手段）
